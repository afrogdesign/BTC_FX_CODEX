# 人とAIの記録連携フロー Obsidian用

更新日: 2026-03-30 JST

この資料は、コードではなく、`あなたと AI がどうやって記録を受け渡しながら作業しているか` を図にしたものです。
対象は次の流れです。

- `Global_BOX` が持つ共通ルール
- 案件フォルダが持つ案件固有の入口
- Obsidian 側の人向けメモ
- `報告して` `AI用に報告して` `人用に報告して` の更新分岐

## 1. まず全体像

```mermaid
flowchart TD
    A["あなた<br>依頼・判断・確認をする"] --> B["AI"]
    B --> C["共通ルールを確認<br>`Global_BOX`"]
    B --> D["案件ルールを確認<br>`AGENTS.md`"]
    B --> E["AIの入口を見る<br>`運用資料/NEXT_TASK.md`"]
    E --> F["次に地図を見る<br>`運用資料/開発ロードマップ.md`"]
    F --> G["足りないときだけ補助資料<br>`運用資料/スレッド引き継ぎファイル.md`<br>`運用資料/progress.md`"]
    G --> H["実作業を進める"]
    H --> I["結果を記録へ戻す"]
    I --> J["AI向け入口更新<br>`NEXT_TASK.md`<br>`開発ロードマップ.md`"]
    I --> K["人向け入口更新<br>`👩‍⚖️秘書.md`"]
    I --> L["人向け履歴更新<br>`📒打ち合わせノート.md`"]
```

## 2. 記録の役割分担

flowchart LR

    B["案件フォルダ"] --> B1["`AGENTS.md`<br>案件ルール"]

    B --> B3["`運用資料/開発ロードマップ.md`<br>AIの地図"]
    B --> B4["`運用資料/progress.md`<br>実施履歴"]
    B --> B5["`運用資料/スレッド引き継ぎファイル.md`<br>例外時メモ"]

    C["Obsidian 側"] --> C1["`👩‍⚖️秘書.md`<br>人の入口"]

## 3. AIが毎回どう入るか

```mermaid
flowchart TD
    A["AI 作業開始"] --> B["まずマシン構成確認<br>`Global_BOX/20_環境情報/マシン構成.md`"]
    B --> C["案件ルール確認<br>`AGENTS.md`"]
    C --> D["AIの入口を読む<br>`運用資料/NEXT_TASK.md`"]
    D --> E["次の大型節目を確認<br>`運用資料/開発ロードマップ.md`"]
    E --> F{"まだ迷う?"}
    F -->|はい| G["例外メモ確認<br>`運用資料/スレッド引き継ぎファイル.md`"]
    F -->|いいえ| H["実作業へ進む"]
    G --> H
    H --> I["必要に応じて `progress.md` や補助資料も確認"]
```

## 4. 人とAIの役割の違い

```mermaid
flowchart TD
    A["あなたの役目"] --> A1["依頼する"]
    A --> A2["優先順位を決める"]
    A --> A3["最終判断する"]

    B["AIの役目"] --> B1["入口を読む"]
    B --> B2["必要な資料を選ぶ"]
    B --> B3["実作業を進める"]
    B --> B4["適切な記録へ戻す"]

    C["記録ファイルの役目"] --> C1["次回の入口になる"]
    C --> C2["人とAIの受け渡しを固定する"]
    C --> C3["履歴と入口を混ぜない"]
```

## 6. 報告指示で何が更新されるか

```mermaid
flowchart TD
    A["あなたが AI に指示する"] --> B{"指示の言い方"}

    B -->|報告して| C["4か所まとめて更新"]
    C --> C1["`運用資料/NEXT_TASK.md`"]
    C --> C2["`運用資料/開発ロードマップ.md`"]
    C --> C3["`👩‍⚖️秘書.md`"]
    C --> C4["`📒打ち合わせノート.md`"]

    B -->|AI用に報告して| D["AI向けだけ更新"]
    D --> D1["`運用資料/NEXT_TASK.md`"]
    D --> D2["必要なら `運用資料/開発ロードマップ.md`"]

    B -->|人用に報告して| E["人向けだけ更新"]
    E --> E1["`👩‍⚖️秘書.md`"]
    E --> E2["`📒打ち合わせノート.md`"]
```

## 7. 実作業のあと、どう記録へ戻すか

```mermaid
flowchart TD
    A["AI が作業完了"] --> B{"何が変わった?"}

    B -->|次の判断が変わった| C["`NEXT_TASK.md` 更新"]
    B -->|フェーズや大型節目が変わった| D["`開発ロードマップ.md` 更新"]
    B -->|人に見せる価値のある変化| E["`👩‍⚖️秘書.md` 更新"]
    B -->|作業がひとまとまり終わった| F["`📒打ち合わせノート.md` 更新"]
    B -->|実施事実を残したい| G["`progress.md` 更新"]
    B -->|例外的で次のAIが迷いそう| H["`スレッド引き継ぎファイル.md` 更新"]
```

## 8. この仕組みの重要ポイント

```mermaid
flowchart TD
    A["この運用の核"] --> B["入口と履歴を分ける"]
    A --> C["人向けとAI向けを分ける"]
    A --> D["共通ルールと案件ルールを分ける"]
    A --> E["短い指示で更新先が決まる"]

    B --> F["AIが毎回迷いにくい"]
    C --> G["人が読んでも重くならない"]
    D --> H["別案件へ流用しやすい"]
    E --> I["報告運用が崩れにくい"]
```

## 9. 一言でまとめると

```text
Global_BOX が土台のルールを持つ
→ 案件フォルダがその案件の入口を持つ
→ AI はまず AI 用入口から入る
→ 人は秘書メモから入る
→ 作業後は「誰向けの記録か」に応じて戻し先を分ける
```

## 10. ファイルごとの超短い意味

- `Global_BOX`
  - 全案件共通の土台ルール
- `AGENTS.md`
  - その案件での運用ルール
- `運用資料/NEXT_TASK.md`
  - AI の今日の入口
- `運用資料/開発ロードマップ.md`
  - AI の地図
- `運用資料/progress.md`
  - 実施履歴
- `運用資料/スレッド引き継ぎファイル.md`
  - 例外時だけの圧縮メモ
- `👩‍⚖️秘書.md`
  - 人の入口
- `📒打ち合わせノート.md`
  - 人の履歴正本
