## 版の考え方

- `Ver02.x`
  - `Phase 0` 観測と `Phase 1` 着手前後の改善版
- `Ver03`
  - `Phase 0` 一周完了と `Phase 1` 中核導入を満たした大型節目
- `Ver04`
  - `Phase 2` の通知品質・安全性強化が live データで動く大型節目
- `Ver05`
  - `Phase 3` の分析基盤と比較モードがそろう大型節目

## 今いちばん重要な判断

- 直近は `Ver02.4-v1` の `Phase 1` 本有効確認を優先し、レビューは AI 事後評価を正本にして人は補助観測へ寄せる
- 通知の見え方は、複数軸の併記より `最終ランク` を主表示にして誤読を減らす
- 本文改善は AI 自由作文より、コード側テンプレート主導を優先する
- 旧版比較より、現行版の `phase1_active=true` / `primary_setup_status=ready` の live 観測を優先する
- `Phase 1` の本有効確認は `Phase 0` 完了前でも前倒しで進め、レビューは最低 5 件を維持すればよい
- `Ver03` は「Ver02.4-v1 の live 観測完了 + 通知後フロー一周完了 + Phase 1 中核導入」がそろうまで名乗らない
