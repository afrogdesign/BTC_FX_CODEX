## 重要判断

- 正式 `trade_execution_gate` は安全基準として維持する。
- `trade_execution_gate=pass` と `paper_orders planned` が出るまでは、実弾 gate を緩和しない。
- ただし前進は厳格 gate 待ちだけにせず、`opportunity_gate` と `paper_positions.csv` の型別成績で進める。
- `opportunity_gate=pass` は実弾候補ではなく、`紙実行候補・実弾不可` として扱う。
- 実弾発注、取引所API送信、秘密鍵連携はまだ行わない。
- スコア値の大きな再調整は、紙ポジションの型別成績と品質ガード後の結果を見てから行う。
- `Phase 1B-lite` は正式 `Phase 1B` ではなく、`SWEEP_WAIT` 限定の専用紙トレード観測レーンとして扱う。

## 新フェーズ

- `Phase A`: 攻め候補の捕捉。`opportunity_gate` を保存し、候補を拾う。
- `Phase B`: ライン監視と再検知。主要ライン接近、flip、failed breakout を追う。
- `Phase C`: 紙の自動実行。pending / opened / closed を管理する。
- `Phase D`: 勝ち筋の選別。PF、TP1先行率、MFE/MAE、取り逃し率、SL偏重、型別の危険条件を見る。
- `Phase E`: 自動取引直前。実弾発注以外の安全装置と運用確認をそろえる。

## 直近の実装方針

- 現在は `Phase D` へ進むための品質ガード実装を優先する。
- `entry / wait / trend_flip` の品質ガードを `src/trade/opportunity_gate.py` に追加する。
- `wait>=80`、`execution<20`、`long + wait>=60 + execution<25`、`long + trend_flip_confirmed_up` を弱い紙候補として分離する。
- `trade_execution_gate` の pass 条件は緩めない。
- `SL/TP` の倍率・距離・RR計算は今回触らない。
- `support_to_resistance_flip` など相対的に有効な market_map 型は残し、候補化条件だけを品質ガードで整える。
- 正式 `paper_orders.csv` と攻め候補の `paper_positions.csv` は分離を維持する。
- 日次レポートで `opportunity_gate`、紙ポジション件数、状態、型別内訳、quality blocker 件数を確認できるようにする。
