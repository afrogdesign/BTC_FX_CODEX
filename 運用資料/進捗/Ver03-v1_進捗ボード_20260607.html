  • 1 つのコマンドで current manual-delivery app state を refresh できる。

  • app-snapshot-status.json から readiness check を再実行できる。
  • app は contract / schema / safety invariants を introspect できる。
  • operator は progress board を開くだけで current app status を把握できる。

  • no send/notify integration
  • no runtime restart/deploy path

  • no paper_positions integration

Ver03-v1 から Ver03-v4 への変化

  • Ver03-v1 は主に redesign / inventory / progress board / orchestration setup の段階だった。
  • Ver03-v2 では implementation と diagnostic/manual-preview の土台を進めた。
  • Ver03-v3 では local/report-only manual-delivery support flow を整備し、人間レビュー前提の local-only 運用に進んだ。
  • Ver03-v4 では public HTML / notification mail / local dashboard を one-source-of-truth で揃える integrated roadmap に進み、manual-trading から future automation までの surface alignment を明示した。
  • その後の current manual-delivery app path では refresh-current-manual-delivery-app --stdout-json と check-current-manual-delivery-app-ready --stdout-json と describe-current-manual-delivery-app-contract --stdout-json が accepted になった。
  • 現在の reviewed baseline は 91dfda6114da890c62f9248fc6dab2e9d9413bbd と 9415a698bb045e2aaf668b9c1eaf9a8d4aee5f70 で表現される current accepted state で、その上に Ver03-v4 の integrated roadmap を重ねている。
  • 現在も live trading や automatic order には進んでいない。
