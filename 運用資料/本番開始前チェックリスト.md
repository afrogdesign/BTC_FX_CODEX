# 本番開始前チェックリスト

このチェックリストは、BTC監視システムを本番運用に入れる前の確認用です。上から順にチェックすれば、見落としを減らせます。

## 1. ファイルと環境の確認
- [ ] `btc_monitor` ディレクトリが手元にある
- [ ] Python が使える
- [ ] Python 3.11以上が理想
- [ ] `requirements.txt` が存在する
- [ ] 依存関係をインストール済み
- [ ] 実行場所を `btc_monitor` に合わせている

確認コマンド例:

```bash
cd /Users/marupro/CODEX/BTC_FX_CODEX/btc_monitor
python3 -m pip install -r requirements.txt
```

## 2. `.env` の本番値確認
- [ ] `OPENAI_API_KEY` が本番用の正しい値になっている
- [ ] `SMTP_USER` が正しい
- [ ] `SMTP_PASSWORD` が正しい
- [ ] `MAIL_FROM` が正しい
- [ ] `MAIL_TO` が正しい
- [ ] ダミー値や仮値が残っていない
- [ ] 本番運用するなら `DRYRUN_MODE=false` にする
- [ ] タイムゾーン設定が意図どおりになっている

補足:
- `DRYRUN_MODE` は「試運転モード」です。`true` のままだと本番動作として不完全になる可能性があります。

## 3. 単発実行チェック
- [ ] `run_cycle()` がエラーなく動く
- [ ] BTCデータ取得まで進む
- [ ] 判定結果が返る
- [ ] `bias` と `confidence` が表示される
- [ ] 途中で例外が出ない

確認コマンド:

```bash
cd /Users/marupro/CODEX/BTC_FX_CODEX/btc_monitor
python3 -c "from pathlib import Path; from config import load_config; from main import run_cycle; cfg=load_config(Path('.')); r=run_cycle(cfg, Path('.')); print(r['bias'], r['confidence'])"
```

## 4. ログ出力チェック
- [ ] [`logs/last_result.json`](/Users/marupro/CODEX/BTC_FX_CODEX/btc_monitor/logs/last_result.json) が更新される
- [ ] `logs/signals/` にシグナル記録が出る
- [ ] 必要なら `logs/csv/trades.csv` に追記される
- [ ] `logs/heartbeat.txt` が更新される
- [ ] `logs/errors/` にエラーが出ていない

補足:
- `heartbeat.txt` は「プログラムが生きているか」を見るための目印です。
- `last_result.json` は「直近の判定結果」です。

## 5. 通知機能チェック
- [ ] SMTP 認証が通る
- [ ] メール送信先が正しい
- [ ] 差出人アドレスが正しい
- [ ] 通知が失敗した場合に保留メール再送の仕組みが動く
- [ ] 本番で通知過多にならない設定になっている

補足:
- SMTP はメール送信サーバーへの接続設定です。
- 認証エラーがあると通知だけ失敗することがあります。

## 6. API連携チェック
- [ ] MEXC API から 4h / 1h / 15m データが取得できる
- [ ] 資金調達率 `funding rate` の取得で落ちない
- [ ] APIタイムアウトや通信エラーが連発していない
- [ ] 実行環境で `urllib3` の LibreSSL 警告が出ても、処理自体は継続できることを把握している

補足:
- 警告は「注意表示」で、必ずしも停止ではありません。
- ただし本番前に、警告と本当のエラーは区別して見ておくと安心です。

## 7. 常駐運用前チェック
- [ ] 単発実行が成功している
- [ ] `.env` が本番設定になっている
- [ ] ログ保存先の容量や権限に問題がない
- [ ] 長時間動かす前提で、PCやサーバーのスリープ設定を確認している
- [ ] 手動停止時の手順を把握している

起動コマンド:

```bash
cd /Users/marupro/CODEX/BTC_FX_CODEX/btc_monitor
python3 main.py
```

## 8. 運用開始後の初回監視ポイント
- [ ] 起動直後に落ちない
- [ ] 数サイクル連続で動く
- [ ] `heartbeat.txt` が継続更新される
- [ ] `last_result.json` が更新され続ける
- [ ] `logs/errors/` に新しい例外が増えていない
- [ ] 通知が意図どおりに飛ぶ、または飛びすぎない

## 9. 最終判定
- [ ] すべてOKなら本番開始
- [ ] 1つでも不安があれば `DRYRUN_MODE=true` で再確認
- [ ] 特にメール送信と `.env` の値は本番前に再確認
