## 状態確認

### 常駐ジョブの状態を見る
```bash
launchctl print gui/$(id -u)/com.afrog.btc-monitor
```

見るポイント:
- `state = running` なら動作中
- `pid = ...` で現在のプロセス番号を確認
- `program = ...` でどの Python で動いているか確認

### 保存された PID ファイルを見る
```bash
cat /Users/marupro/CODEX/BTC_FX_CODEX/btc_monitor/logs/runtime/monitor.pid
```

見るポイント:
- `launchctl print ...` の `pid = ...` と一致していれば正常

### 実際に動いているプロセスを短く見る
```bash
pgrep -fl "/Users/marupro/CODEX/BTC_FX_CODEX/btc_monitor/main.py"
```

## ログ確認

```bash
tail -n 50 /Users/marupro/CODEX/BTC_FX_CODEX/btc_monitor/logs/runtime/monitor.out
```

### エラーログを見る


## 再起動

### 設定変更後に再起動する
```bash
cd /Users/marupro/CODEX/BTC_FX_CODEX/btc_monitor
zsh tools/start_monitor.sh
```

用途:
- `.env` を変更したあとに新しい設定を反映したいとき
- 常駐ジョブを入れ直したいとき

再起動後の確認:
```bash
launchctl print gui/$(id -u)/com.afrog.btc-monitor
```

## 停止

### いま動いている常駐を停止する
```bash
launchctl bootout gui/$(id -u) ~/Library/LaunchAgents/com.afrog.btc-monitor.plist
```

停止後の確認:
```bash
launchctl print gui/$(id -u)/com.afrog.btc-monitor
```

補足:
- エラー表示になれば停止できています
- また動かすときは [再起動](#再起動) のコマンドを使います

## 自動起動の確認

### 自動起動設定ファイルの内容を見る
```bash
plutil -p ~/Library/LaunchAgents/com.afrog.btc-monitor.plist
```

見るポイント:
- `RunAtLoad => 1`
  - ログイン時や読み込み時に起動する設定
- `KeepAlive => 1`
  - 落ちたときに再起動しやすくする設定

### launchd に登録されている状態を見る
```bash
launchctl print gui/$(id -u)/com.afrog.btc-monitor
```

## 自動起動を止める・戻す

### 自動起動も含めて止める
```bash
launchctl bootout gui/$(id -u) ~/Library/LaunchAgents/com.afrog.btc-monitor.plist
launchctl disable gui/$(id -u)/com.afrog.btc-monitor
```

意味:
- `bootout`: 今動いているものを外す
- `disable`: 次回ログイン後も自動起動しないようにする

### 自動起動を再開する
```bash
launchctl enable gui/$(id -u)/com.afrog.btc-monitor
cd /Users/marupro/CODEX/BTC_FX_CODEX/btc_monitor
zsh tools/start_monitor.sh
```

## 補足

この監視は「メールが来たら即売買する」前提ではなく、「相場の変化に気づく」ための補助です。
通知が増えすぎると感じたら、`.env` の `CONFIDENCE_ALERT_CHANGE` を `8` か `9` に戻すと調整しやすいです。
