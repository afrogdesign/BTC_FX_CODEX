## 状態確認

### Ver02 本番常駐の状態を見る
```bash
ssh marupro@192.168.1.38
launchctl print gui/$(id -u)/com.afrog.btc-monitor-ver02
```

見るポイント:
- `state = running` なら動作中
- `pid = ...` で現在のプロセス番号を確認
- `program = ...` でどの Python で動いているか確認

### Ver01 本番常駐の状態を見る
```bash
ssh marupro@192.168.1.38
launchctl print gui/$(id -u)/com.afrog.btc-monitor-ver01
```

### 開発環境で常駐が残っていないか確認する
```bash
launchctl list | grep -E 'com\.afrog\.btc-monitor|ver02'
```

見るポイント:
- 何も出なければ、ローカル常駐は残っていない
- `com.afrog.btc-monitor` が出る場合は、意図せず常駐が残っている可能性がある

### Ver02 本番の PID ファイルを見る
```bash
ssh marupro@192.168.1.38
cat /Users/marupro/CODEX/BTC_FX_CODEX_ver02/btc_monitor/logs/runtime/monitor.pid
```

見るポイント:
- `launchctl print ...` の `pid = ...` と一致していれば正常

### 本番機で実際に動いている Ver02 プロセスを短く見る
```bash
ssh marupro@192.168.1.38
pgrep -fl "/Users/marupro/CODEX/BTC_FX_CODEX_ver02/btc_monitor/main.py"
```

## 再起動

### Ver02 本番を再起動する
```bash
ssh marupro@192.168.1.38
cd /Users/marupro/CODEX/BTC_FX_CODEX_ver02/btc_monitor
```

```bash
ssh marupro@192.168.1.38
launchctl print gui/$(id -u)/com.afrog.btc-monitor-ver02 | grep -E 'state =|pid ='
```

## 自動起動の確認

### Ver02 本番の自動起動設定ファイルを見る
```bash
ssh marupro@192.168.1.38
plutil -p ~/Library/LaunchAgents/com.afrog.btc-monitor-ver02.plist
```

見るポイント:
- `RunAtLoad => 1`
  - ログイン時や読み込み時に起動する設定
- `KeepAlive => 1`
  - 落ちたときに再起動しやすくする設定

### launchd に登録されている状態を見る
```bash
ssh marupro@192.168.1.38
launchctl print gui/$(id -u)/com.afrog.btc-monitor-ver02
```

## 自動起動を止める・戻す

### Ver02 本番の自動起動も含めて止める
```bash
ssh marupro@192.168.1.38
launchctl bootout gui/$(id -u) ~/Library/LaunchAgents/com.afrog.btc-monitor-ver02.plist
launchctl disable gui/$(id -u)/com.afrog.btc-monitor-ver02
```

意味:
- `bootout`: 今動いているものを外す
- `disable`: 次回ログイン後も自動起動しないようにする

### Ver02 本番の自動起動を再開する
```bash
ssh marupro@192.168.1.38
launchctl enable gui/$(id -u)/com.afrog.btc-monitor-ver02
cd /Users/marupro/CODEX/BTC_FX_CODEX_ver02/btc_monitor
zsh tools/start_monitor_ver02_prod.sh
```

## 補足

この監視は「メールが来たら即売買する」前提ではなく、「相場の変化に気づく」ための補助です。
通知が増えすぎると感じたら、`.env` の `CONFIDENCE_ALERT_CHANGE` を `8` か `9` に戻すと調整しやすいです。
