Ver02.3 評価システムの見方

この資料は「いまの BTC 監視システムが、何を見て、どういう順番で、なぜ ready / watch / invalid を決めているか」を、人が判断できる形にしたものです。 目的は、メールを見たときに「今は攻め場なのか、待つべきなのか、そもそも触らない方がいいのか」を体感で比較しやすくすることです。

まず全体像

Ver02.3 は、単純にシグナルを増やすための緩和ではありません。方向判定は少し早めに出しつつ、位置が悪いところでの飛び乗りは別レイヤで止める設計です。

1. Score 4時間足、1時間足、15分足、出来高、RSI、breakout などからロング / ショートの点数差を出します。
2. Confidence 3つの時間足が揃っているか、RR が十分か、危険フラグが多すぎないかで信頼度を 0〜100 に整えます。
3. Setup entry zone の中か、近いか、trigger があるかで ready / watch / invalid を決めます。
4. Position Risk 流動性、板、清算クラスター、未 sweep を見て「入る位置の悪さ」を別評価します。
5. Tier 条件がかなりそろうと strong_machine、AI も一致すると strong_ai_confirmed になります。
6. Notify bias 変化、setup 昇格、prelabel 改善、tier 昇格など、意味のある変化が出た時だけ通知します。

Ver02.3 の狙い

  • 取りこぼしを減らす: breakout と trigger の扱いを見直し、方向が出ているのに wait へ寄りすぎる問題を緩めています。
  • 雑な飛び乗りは止める: 位置リスクを別評価し、方向は合っていても場所が悪ければ watch や invalid へ落とします。
  • 短期の勢いを少し拾う: 15分足シグナル、breakout、出来高拡大を前より効かせています。

Ver02.3 でまだ守っていること

  • ショートは依然として慎重: bias の差分閾値も confidence 最低値も、ショート側の方が厳しめです。
  • Funding と ATR は止める材料: 禁止域や極端値は、いまもかなり強いブレーキです。
  • ready でも場所は重要: prelabel が悪いと、機械的な ready が watch / invalid に落ちます。

1. Score の仕組み

ここでは「相場の方向」を決めます。ロング点数とショート点数を別々に作り、差分が大きい方を bias にします。最後は 0〜100 表示へ正規化されます。

要素 ロング / ショートへの効き方 意味
4時間足トレンド 上昇基調 +15 / 下降基調 +15 大きな流れを最優先で見る部分です。
EMA 配列 強気 +12 / 弱気 +12 移動平均の並びがきれいかを見ています。
EMA20 の傾き 上向き +6 / 下向き +6 トレンドの勢いが続いているかを見る補助です。
4時間足構造 hh_hl +14 / lh_ll +14 高値安値の切り上げ / 切り下げを強く評価します。
1時間足構造 hh_hl +12 / lh_ll +12 中位足も同じ方向なら追い風です。
support / resistance 近接 support 近い long +7 / resistance 近い short +7 押し目・戻り売りの足場として見ています。
breakout 上抜け +10 / 下抜け +10 現在足を除く過去20本を抜けた時だけ効きます。
出来高拡大 両方向 +4 15分足の出来高比が 1.15 以上で勢い材料になります。
15分足シグナル 一致方向 +3 直近の短期方向を少しだけ上乗せします。
transition 補助 転換帯で上向き +6 / 下向き +6 トレンド転換のヒントを軽く拾います。

Score のブレーキ

ブレーキ要素 減点 意味
逆側ゾーンが近い -6 ロングなのに resistance 直下、ショートなのに support 直上の時です。
RR 不足 -8 利幅より損切り幅が重すぎる時です。
レンジ中央 -10 上にも下にも伸びにくい真ん中を嫌います。
ATR 極端 -6 値動きが荒すぎる / 眠すぎる時です。
Funding warning -4 表示だけでなく、今は raw score にも直接効きます。
Funding prohibited -12 かなり強い逆風として扱います。
bias 判定の現在値は、ロング差分 +8 以上で long、ショート差分 -9 以下で short、それ以外は wait です。

2. Confidence の仕組み

Confidence は「その方向をどれだけ信じるか」です。Score をそのまま使うのではなく、時間足一致、phase、RR、危険フラグを加味して 0〜100 にします。

項目 現在の効き方
3時間足一致 3つ一致 +15、2つ一致 +8
market regime トレンド +10、range -5、volatile -10
phase breakout +6、trend_following +5、pullback +3、range -5、reversal_risk -10
RR 2.0以上 +10、1.5以上 +5、1.3未満から減点、1.1未満は -18
反対側までの余白 1.5ATR以上 +5、0.8ATR未満 -5
Critical zone -10

warning は major / minor に分かれます。major は Funding 禁止、ATR extreme、Critical zone。minor は Funding warning や位置リスク由来の注意です。合計減点は最大 -18 に抑えています。

現在の confidence 最低ライン

方向 最低ライン 見方
long 45 やや早めに拾う設定です。
short 55 ロングより慎重です。下方向は一段強い裏付けを要求しています。

3. Setup の仕組み

Setup は「今すぐ候補か、まだ待つか」を決める層です。entry zone と trigger の両方を使います。

状態 条件 人間の見方
ready entry zone 内で trigger がある、または zone まで 0.08 ATR 以内で trigger がある 「候補としてかなり近い」です。メール比較でまず見たいのはここです。
watch entry zone まで 0.3 ATR 以内だが trigger 不足、または場所は近いが詰め切れていない 方向は見えているが、まだ飛びつかない方がいい状態です。
invalid RR 不足、ATR 異常、Funding 禁止、confidence 不足、warning 3件以上など いま触らない方がいい状態です。
trigger は現在、上方向なら breakout_up または 15分足の出来高比 1.15 以上、下方向なら breakout_down または出来高比 1.15 以上です。

4. Position Risk の仕組み

ここは Ver02.3 の重要部分です。方向が合っていても「入る場所が悪い」なら強く止めます。

見るもの 今の考え方
上側 / 下側の流動性 近すぎると、その方向に走る前に一度刈られやすいと見ます。
板の壁 0.45 ATR 以内はかなり近い壁、0.9 ATR 以内は注意帯です。
清算クラスター ロングなら下側クラスター、ショートなら上側クラスターを重く見ます。逆側は 0.3 倍で軽く扱います。
未 sweep 最近 sweep 済みなら -12 の追い風、未 sweep なら +6 の不安要素です。
OI / CVD / 板偏り 偏りが逆風ならさらに加点され、location risk が上がります。

prelabel の意味

prelabel location risk 人間の解釈
ENTRY_OK 33 未満 場所はかなり良いです。ready なら前向きに比較対象です。
RISKY_ENTRY 33〜55 未満 方向は合っていても、今入るには少し雑です。ready でも watch に落ちることがあります。
SWEEP_WAIT 55〜80 未満 一度振ってからの反応待ちです。今の Ver02.3 が特に注意して止める帯です。
NO_TRADE_CANDIDATE 80 以上 方向よりも場所が悪すぎるので、今は触らない判断です。

5. signal tier の見方

signal tier は「ただの候補」なのか「かなり条件が揃った候補」なのかを見分けるための追加ラベルです。

tier 条件 意味
normal 通常状態 まずは本文を見て判断する通常メールです。
strong_machine ENTRY_OK、ready、最低 confidence +10、RR 1.8以上、重大 no_trade なし、major warning 0、minor warning 1 以下、機械一致 かなり条件が揃った機械評価です。
strong_ai_confirmed strong_machine に加えて AI も同方向、confidence 0.65 以上、quality A/B 機械と AI の両方が強く揃った状態です。

6. どんな時にメールが来るか

  • bias が wait → long / short に変わった時: 方向が初めて出た時です。
  • setup が watch / invalid → ready に昇格した時: いちばん重要な変化です。
  • prelabel が改善した時: SWEEP_WAIT → RISKY_ENTRY → ENTRY_OK のような改善です。
  • confidence が大きく変わった時: 現在は 10 以上のジャンプで拾います。
  • signal tier が昇格した時: normal → strong_machine → strong_ai_confirmed です。
いまの観測では、本番 MBP2020 側は [Ver02.1] [CLI]、iMac 側は [Ver02.3-OBS] [CLI] の件名で届きます。見比べる時は、まずこのラベルで版を区別します。

どこをどう観察するとよいか

まずはメール件名だけで比較する

  • Ver02.3-OBS の方が bias を早く出すか: 同じ時間帯で wait から抜けるタイミングを見ます。
  • ready が増えすぎていないか: 増えるのは狙いどおりですが、雑な ready が増えすぎると危険です。
  • ショートの出方: Ver02.3 は short を完全に緩めたわけではないので、出方が極端に多くなっていないか見ます。

次に本文で見るポイント

観察項目 見る理由
primary_setup_status 最重要です。Ver02.3 が watch 止まりを ready に押し上げたかを見ます。
prelabel 方向が良くても位置が悪い時は止められているかを確認できます。
signal_tier 「ただの候補」か「かなり条件が揃った候補」かを一段で見分けられます。
warning_flags / risk_flags なぜ watch / invalid になったかの理由を読み解けます。
phase pullback 狙いか breakout 狙いかで、期待する値動きの性格が変わります。

ログで見るならここ

  • logs/runtime/monitor.out: 本体の通常実行ログ
  • logs/runtime/monitor.err: 実行エラー、CLI / API 失敗、SMTP 失敗など
  • logs/heartbeat.txt: サイクルが回っているか
  • logs/last_result.json: 最後の判定結果を生で確認したい時
  • logs/last_notified.json: 実際に通知された内容

今の Ver02.3 が向いている取引スタイル

取引スタイル 相性 理由
押し目買い / 戻り売り 高い support / resistance と entry zone を強く見ているので、ゾーン回帰型と相性が良いです。
breakout 追随 中〜高 Ver02.3 で breakout と出来高 trigger を拾いやすくしました。ただし位置リスクが悪いと止まります。
流動性 sweep 後の反転取り 中 SWEEP_WAIT を持っているため、振った後の反応を見る前提とは相性があります。
超短期スキャルプ 低め 4時間足と1時間足をかなり重視するので、数分単位の瞬発だけを取る用途には重いです。
逆張りレンジ中央エントリー 低い レンジ中央ははっきり減点されるので、このスタイルには向いていません。
ナンピン前提の裁量 低い 今の設計は「悪い位置では最初から入らない」思想なので、平均化前提とは噛み合いません。

いま期待したい変化

  • Ver02.1 では watch 止まりだった場面が、Ver02.3 では ready へ上がること
  • bias が wait から抜けるタイミングが少し早くなること
  • それでも prelabel=SWEEP_WAIT や NO_TRADE_CANDIDATE が適切に残ること

警戒したい変化

  • ready が急増しすぎて、本文の warning が多いのにメールだけ強気になること
  • RISKY_ENTRY や SWEEP_WAIT が減りすぎて、位置リスクが効いていないこと
  • ショートが不自然に増えて、下方向の判定が雑になっていること

最後に

このシステムは「未来を当てる機械」ではなく、「いま攻める価値があるかを層ごとに絞り込むフィルター」です。 Ver02.3 の価値は、単にシグナル数が増えることではなく、取りこぼしを減らしつつ、悪い場所はまだ止めているか にあります。 しばらくは Ver02.1 と Ver02.3-OBS のメールを並べて、体感差を見るのが最も分かりやすい観察方法です。
