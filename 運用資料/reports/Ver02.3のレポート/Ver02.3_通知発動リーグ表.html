通知発動リーグ表

いまのリーグ表は 最終ランク を基準に読みます。補足状態はランクではなく説明用です。したがって「どのメールが来るか」は、この表の最終ランク列だけ見れば判断できます。

最終ランクの5段階

  • 🔥 強い本通知
  • 🟠 高め本通知
  • 📊 通常の本通知
  • 👀 注意報
  • ⚪ 送信なし

送信あり / なし

送信あり: 🔥 / 🟠 / 📊 / 👀 送信なし: ⚪

旧ラベルの 執行可 / 監視 / 無効 / 注意報 / 中立 は、この判定を上書きしません。

強い本通知の条件

signal_tier = strong_machine のときだけ 🔥 になります。

  • prelabel = ENTRY_OK
  • primary_setup_status = ready
  • confidence が方向下限 + 10 以上
  • rr_estimate ≥ 1.8

最終ランク決定表

最終ランク 決定条件 メール 補足状態の例
🔥 強い本通知 notification_kind = main かつ signal_tier = strong_machine 送る 執行可 が典型
🟠 高め本通知 main で、ready / ENTRY_OK / confidence 高め / RR改善 / score_gap 大 のどれか 送る 監視、執行可、無効 のいずれもありうる
📊 通常の本通知 main だが、上の強化条件までは未達 送る 無効 や 監視 が典型
👀 注意報 notification_kind = attention 送る 補足状態も 注意報
⚪ 送信なし notification_kind != main/attention、または bias/変化/条件不足 送らない 中立 など

補足状態の位置づけ

補足状態 意味 最終ランクとの関係
執行可 条件成立 🔥 / 🟠 / 📊 のどれにも付きうる
監視 条件接近 🟠 や 📊 と組み合わさりやすい
無効 執行不可 📊 や 🟠 で出ることがある
注意報 方向変化の早期共有 最終ランク 👀 の中身説明
中立 方向優位なし 最終ランク ⚪ の中身説明
