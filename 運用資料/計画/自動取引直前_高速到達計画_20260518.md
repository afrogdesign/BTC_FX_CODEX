# 自動取引直前 高速到達計画

更新日: 2026-05-18 JST

## 結論

自動取引の直前までは、正式 `Phase 1B` の発生を待たずに進める。
実弾発注はまだ禁止したまま、`watch` / `Phase 1A` / `Phase 1B-lite` / `market_map` の有望候補を `opportunity_gate` で拾い、紙ポジションとして自動記録する。

目的は、スコアの細かい再調整より先に「候補捕捉」「紙実行」「出口判定」「型別成績」をそろえ、勝てるラインと逃しているチャンスを早く見つけること。

## 現在の前提

- 現在版: `Ver02.5-v8`
- 既存実装: `position_sizing`、`exit_manager`、`trade_execution_gate`、`phase1_observation_gate`、`phase1b_lite_gate`、`market_map`、AI事後評価、日次レポート。
- 直近課題: `trade_execution_gate=pass` と `paper_orders planned` が 0 件で、厳格 gate 待ちだけでは前進が遅い。
- 新方針: 正式 gate は残しつつ、別系統の紙実行候補を増やす。

## 新しい実装方針

1. `opportunity_gate` を追加する。
   - 対象は `phase1_observation_gate=pass`、`phase1b_lite_gate=pass`、有効な `market_map` 型。
   - hard no-trade、Funding禁止、ATR極端、data quality 不良は除外する。
   - `trade_execution_gate=pass` は最上位候補として扱うが、必須条件にはしない。

2. `paper_positions.csv` を追加する。
   - 保存先: `logs/csv/paper_positions.csv`
   - 役割: 自動取引直前の紙ポジション台帳。
   - `ready` は `opened`、それ以外の有望候補は `pending` として記録する。
   - 正式 `paper_orders.csv` とは混ぜない。

3. 通知表示を分離する。
   - 正式 gate 通過のみ `執行候補`。
   - `opportunity_gate=pass` は `紙実行候補・実弾不可`。
   - ユーザーが実弾候補と誤読しない文言にする。

4. 日次レポートへ追加する。
   - `opportunity_gate=pass`
   - `paper_positions` 件数
   - `pending/opened/closed` 状態
   - `exit_status` 内訳
   - `opportunity_type` 内訳
   - closed 件数、勝率、平均R、簡易PF
   - missed_opportunity と古い pending
   - 未記録の欠落件数

## Phase B-C: 紙ポジション状態遷移

`paper_positions.csv` は候補台帳ではなく、実弾なしの紙ポジション管理として扱う。
既存列は維持し、`opened_at_jst`、`closed_at_jst`、`exit_status`、`exit_reason`、`close_price`、`tp1_hit_at_jst`、`tp2_hit_at_jst`、`sl_hit_at_jst`、`timeout_at_jst`、`mfe_atr`、`mae_atr`、`realized_r`、`missed_opportunity`、`missed_reason`、`last_evaluated_at_utc` を追加する。

- `position_status` は `pending` / `opened` / `closed` の 3 値。
- `opportunity_gate=pass` を `signal_id` で upsert し、既存行の状態は破棄しない。
- `ready` は即 `opened`、それ以外は `pending`。
- `pending` は entry price に触れたら `opened`。
- entry に触れないまま TP 方向へ走ったら `closed / missed_opportunity`。
- `opened` は TP2 / SL / timeout で `closed`。
- 判定は 15分足の `high` / `low` を使う。long は entry/tp が `high >= price`、SL が `low <= stop`。short は entry/tp が `low <= price`、SL が `high >= stop`。
- 同一 candle で TP と SL が両方成立した場合は、保守的に SL 優先。
- TP1 到達後、`breakeven_after_tp1=true` なら実効SLを entry price へ引き上げる。
- timeout は `opened_at + timeout_hours`、未設定時は 12h。
- pending は signal から 24h 未到達なら `closed / entry_not_reached`。
