# ロング誤判定と下落取り逃し改善計画

更新日: 2026-04-30 JST

## 目的

`2026-04-29 19:05 JST` の通知 `20260429_100500` のように、ロング推奨に見える通知が実際には大きな下落になった失敗を、原因と改善手順に分けて整理する。

今回の主目的は、すぐにショート実行へ反転させることではない。まず「ロングを打つべき」と誤読される通知を止め、次に同型の下落警戒を検出し、最後にショート観測候補として追えるようにする。

## 代表例

- signal: `20260429_100500`
- 時刻: `2026-04-29T19:05:00+09:00`
- 件名: `高め本通知 / 上方向バイアス / 流動性回収が未完了`
- 価格: `77,605.5`
- bias: `long`
- score: `long=92 / short=50 / gap=42`
- setup: `primary_setup_side=long / status=watch / reason=entry_zone_not_reached`
- gate: `trade_execution_gate=blocked`
- 通知理由: `confidence_jump`, `prelabel_improved`
- risk flag: `sweep_incomplete`
- 24時間後評価: `direction_outcome=wrong`, `entry_outcome=poor_entry`
- MFE/MAE: `MFE24h=1.6813 ATR`, `MAE24h=12.2717 ATR`

内部判定では `watch` かつ `trade_execution_gate=blocked` だったため、実行候補ではなかった。問題は、件名と本文が人間には「ロング推奨」に見えたことと、下方向転換の兆候がスコア上で十分に重く扱われなかったこと。

## 失敗原因

### 1. 文面が実行推奨に見えた

`primary_setup_status=watch`、`trade_execution_gate=blocked` なのに、件名が `高め本通知` と `上方向バイアス` を先に出していた。

本文には `執行判断: 監視継続` があるが、件名と冒頭の印象が強く、ロング実行を推奨しているように見える。

改善対象:

- `src/presentation/sanitize.py`
- `src/ai/summary.py`
- `tests/test_summary_format.py`
- `tests/test_summary_wording.py`

### 2. 上位足ロング要因が強く残りすぎた

19:05 の top positive factors は以下だった。

- `ema_alignment_bullish=12`
- `structure_1h_hh_hl=12`
- `breakout_up=10`

一方で、下方向要因は `transition_direction=down` による `transition_down_hint=6` 程度で、ロング優勢を反転させるほど強くなかった。

改善対象:

- `src/analysis/scoring.py`
- `tests/test_eval_rebalance.py`

### 3. 失敗ブレイク型を独立検出していない

この例は `breakout_up` としてロング方向に加点されたが、実際には再検討帯に届かず大きく下落した。

`breakout_up` 後に、下側流動性未回収、転換帯、反対方向への推移、MAE 拡大が重なる型を、`failed_breakout_down_reversal` として別扱いにする必要がある。

改善対象:

- `tools/log_feedback.py`
- `tests/test_log_feedback.py`

### 4. ショート候補化が遅い

19:05 の時点ではショートも watch だったが、通知上はロング側だけが主役になった。

その後、01:05 に `short` 側の注意報が出ている。つまり、ショート方向への切り替えは検出できているが、初動の下落警戒としては遅い。

改善対象:

- `src/analysis/scoring.py`
- `src/presentation/sanitize.py`
- `tools/log_feedback.py`

## 改善方針

### 優先度 1: 誤読防止

`watch` かつ `trade_execution_gate=blocked` の本通知は、件名と冒頭で実行不可を明示する。

実装指示:

- 件名に `監視` または `実行不可` を含める。
- `高め本通知` を出す場合でも、`上方向バイアス` 単独ではなく `上方向監視` のようにする。
- 本文冒頭に `これは実行候補ではない` と読める行を追加する。
- `ENTRY_OK + watch + entry_zone_not_reached` は `位置条件は悪くない` ではなく、`位置は悪くないが未到達` と表示する。

受け入れ条件:

- `20260429_100500` 相当の fixture で、件名がロング実行推奨に見えない。
- 本文冒頭で `trade_execution_gate=blocked` 相当の意味が分かる。
- `ready` の本当の実行候補では、既存の強い表現が維持される。

### 優先度 2: long_reversal_risk の追加

ロング優勢でも下落警戒を出す条件を追加する。

候補条件:

- `bias=long`
- `market_regime=transition`
- `transition_direction=down`
- `primary_setup_status=watch`
- `primary_setup_reason=entry_zone_not_reached` または `confidence_below_min`
- `sweep_incomplete` を含む
- `near_resistance_penalty`、`orderbook_ask_heavy`、`ask_wall_close`、`lower_liquidity_close`、`long_flush_exhaustion` のいずれかを含む

実装指示:

- 上記条件を満たす場合、`risk_flags` または warning 側へ `long_reversal_risk` を追加する。
- `long_reversal_risk` がある場合、long の最終ランクを一段下げる。
- `long_reversal_risk` がある場合、通知理由に `下落警戒` を出す。

受け入れ条件:

- `20260429_100500` 相当で `long_reversal_risk` が出る。
- 通常の uptrend long ready では `long_reversal_risk` が出ない。
- `long_reversal_risk` がある本通知は `高め本通知` ではなく、原則 `注意報` または `通常の本通知` へ落ちる。

### 優先度 3: counter_long_short_watch の追加

ロング方向が強いが、実際には下落の初動になりやすい型を、ショート観測候補として記録する。

実装指示:

- `long_reversal_risk` があり、short setup が `watch` 以上なら `counter_long_short_watch` として Phase 1A 観測候補へ出す。
- これは `trade_execution_gate=pass` ではなく、あくまで観測用にする。
- `observation_paper_orders.csv` には実注文ではなく観測として保存する。

受け入れ条件:

- `20260429_100500` 相当が `counter_long_short_watch` 候補として集計される。
- `paper_orders.csv` には入らない。
- daily-sync で件数と代表例が確認できる。

### 優先度 4: failed_breakout_down_reversal レポート

同型失敗を定量的に追うレポートを追加する。

集計条件:

- `bias=long`
- `phase=breakout`
- `breakout_up` または `top_positive_factors` に `breakout_up`
- `primary_setup_status=watch`
- `direction_outcome=wrong`
- `entry_outcome=poor_entry`
- `signal_based_MAE_24h >= 5.0`

出力内容:

- 件数
- 代表 signal
- 平均 MFE/MAE
- 主な risk_flags
- 主な notify_reason
- `long_score / short_score / score_gap`
- 次に触るべき候補ファイル

受け入れ条件:

- `20260429_100500` が代表例に出る。
- `daily-sync` または専用 CLI で再生成できる。
- 次回以降、同型が増えたかを比較できる。

## 判定基準

- 成功: `20260429_100500` 型が「ロング実行推奨」ではなく「上方向監視だが下落警戒」と読める。
- 成功: 同型失敗が daily-sync または専用レポートで継続追跡できる。
- 成功: `trade_execution_gate=pass` は 0 件のままでも、ショート観測候補を Phase 1A として分離して追える。
- 失敗: 件名に `上方向バイアス` と `高め本通知` だけが強く残り、人間がロング実行推奨と読める。
- 失敗: 通常の強い long ready まで過剰に抑制する。

## 注意点

- `Phase 1B` の gate はまだ緩めない。
- ショート化は実注文ではなく、まず観測候補として扱う。
- `long_reversal_risk` は下落警戒であり、単独でショート実行を意味しない。
- 既存の `confidence_watch_learning` と混同しない。今回の主対象は、`breakout_up` 後に大きく下落した失敗型。
