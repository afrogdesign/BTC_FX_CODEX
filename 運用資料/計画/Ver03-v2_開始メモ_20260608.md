# Ver03-v2 開始メモ

## 1. 目的

`Ver03-v1` は、計画フォルダ再構成までを完了した節目として固定する。

`Ver03-v2` は、再構成済みの計画フォルダを起点に、実用的な BTC トレード支援システムへ向けて次の段階を進めるブランチである。

## 2. 分岐元

- base branch: `Ver03-v1`
- base commit: `e3506e4 Rebuild Ver03-v1 planning folder`
- new branch: `Ver03-v2`

## 3. Ver03-v2 の役割

`Ver03-v2` では、以下を順番に進める。

1. `NEXT_TASK.md` を新しい計画入口へ同期する。
2. AI が最初に読むファイルを整理する。
3. 実装ロードマップに基づいて、次の実装対象を1つずつ決める。
4. Active Plan candidate の intraperiod 検証へ進む前に、入力CSV・OHLCV・出力CSV・report の境界を固定する。
5. メール通知で人間が実際に使える行動計画を優先する。
6. 将来自動化へ進むための検証ログを整備する。

## 4. Ver03-v2 でまだやらないこと

- 実弾発注 API は追加しない。
- 取引所 API キーや秘密鍵は扱わない。
- 自動注文送信はしない。
- `ACTIVE_*` を正式GOとして扱わない。
- `trade_execution_gate=pass` と `ACTIVE_*` を混同しない。
- intraperiod 検証が安定するまで、既存 `paper_positions.csv` に Active Plan 候補を混ぜない。
