# BTC Monitor 新マイルストーン定義

更新日: 2026-05-26 JST

## 基本方針

旧設計の「正式 Phase 1B が出るまで待つ」方針をやめ、自動取引直前までの準備を先に進める。
実弾発注はまだしないが、紙の自動実行・追跡・成績評価までは早期に完成させる。

2026-05-26 時点では、Phase A-C の骨格は実装済みで、現在は `Phase C` 観測中かつ `Phase D` 初期段階である。
次は `entry / wait / trend_flip` 品質ガードを実装し、勝てる型と危険な型を分ける。

## Phase A: 攻め候補の捕捉

目的: チャンス候補を逃さず拾う。

完了条件:

- `opportunity_gate` / `opportunity_type` / `opportunity_reasons` が保存される。
- `Phase 1A`、`Phase 1B-lite`、有効な `market_map` 型を紙実行候補にできる。
- hard no-trade と data quality 不良は除外できる。

現在状態:

- 実装済み。
- `opportunity_gate` は shadow / report に流れている。
- ただし `opportunity_gate=pass` は実弾候補ではない。

## Phase B: ライン監視と再検知

目的: 入るべきラインに近づいた候補を追跡する。

完了条件:

- 主要サポート/レジスタンス、flip、failed breakout、entry zone 接近を追える。
- pending 候補を次回サイクル以降も確認できる。
- チャンスが entry に来たか、来ずに走ったかを判定できる。

現在状態:

- 実装済み・観測中。
- `market_map`、flip、failed breakout、trend_flip 系の記録が入り、`market_map_effectiveness` で評価中。
- `support_to_resistance_flip` は相対的に有効だが、候補化時の entry / wait 条件がまだ粗い。

## Phase C: 紙の自動実行

目的: 実弾なしで、注文計画から出口までを自動化する。

完了条件:

- `paper_positions.csv` が作られる。
- pending / opened / closed の状態管理ができる。
- TP / SL / timeout が自動判定される。
- 正式 `paper_orders.csv` と攻め候補の紙ポジションが分離されている。

現在状態:

- 実装済み・観測中。
- `paper_positions.csv` は `pending -> opened -> closed` を扱う。
- TP2 / SL / timeout / missed_opportunity / entry_not_reached の後追い評価が動いている。
- ただし `sl_hit` 偏重が残っているため、Phase D の品質選別が必要。

## Phase D: 勝ち筋の選別

目的: 勝てる型と危険な型を分ける。

完了条件:

- `opportunity_type` 別に PF、勝率、TP1先行率、MFE/MAE を見られる。
- missed opportunity を集計できる。
- `sl_hit` 偏重を entry / wait / execution / side / market_map flag 別に切り分けられる。
- 有効な型だけを gate / score / entry / SLTP 調整対象にできる。

現在状態:

- 初期段階。
- `paper_opportunity_diagnostics_20260526.md` と `paper_entry_sl_wait_redesign_20260526.md` で、型別PF、SL偏重、long / wait / execution の弱点を分離中。
- 直近の active spec は `chatgpt/specs/active/20260526_entry_wait_trend_flip_quality_guard.md`。
- 現在の重点は、候補を増やすことではなく、弱い紙候補を品質ガードで分離すること。

## Phase E: 自動取引直前

目的: 実弾発注以外の運用部品をそろえる。

完了条件:

- 発注直前の注文計画、サイズ、出口、blackout、kill switch、日次成績確認がそろう。
- 実弾発注APIはまだ無効。
- ユーザーが明示許可するまで、取引所への注文送信は実装しない。

現在状態:

- 未到達。
- `trade_execution_gate=pass` と `paper_orders planned` が自然発生していない。
- kill switch / blackout / 実弾直前の安全装置はまだ未整備。

## 禁止事項

- 実弾発注を送信しない。
- 取引所API送信を実装しない。
- 秘密鍵や取引所APIキーを新しく扱わない。
- `opportunity_gate=pass` を `執行候補` と表示しない。
- `Phase 1B-lite` を正式 `Phase 1B` と扱わない。
- 成績が出る前にスコアを大きく緩めない。
- quality blocker 前に SL/TP 倍率を広げない。
