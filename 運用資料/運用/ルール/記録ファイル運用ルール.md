# 記録ファイル運用ルール

更新日: 2026-04-17 JST

## 読む順

1. `運用資料/NEXT_TASK.md`
2. `運用資料/開発ロードマップ.md`
3. 必要なときだけ `運用資料/履歴/スレッド引き継ぎファイル.md`
4. 実施履歴が必要なときだけ `運用資料/履歴/progress.md`

## 役割

- `NEXT_TASK.md`
  - 直近の判断と次の作業
- `開発ロードマップ.md`
  - フェーズと大型節目
- `履歴/progress.md`
  - 実施履歴
- `履歴/スレッド引き継ぎファイル.md`
  - 例外用
- Obsidian 側 `👩‍⚖️秘書.md`
  - 人向け入口。最新状態だけを短く残す
- Obsidian 側 `📒打ち合わせノート.md`
  - 人向け履歴

## `👩‍⚖️秘書.md` の固定フォーマット

`👩‍⚖️秘書.md` は毎回この形にそろえる。

```md
## 最新状態

- 主系統、最新コミット、デプロイ状態
- 最新レポートの主要数値
- 現在の判断
- 直近で追加された仕組みや注意点

## 次に見る

- 次に確認すること 1
- 次に確認すること 2
- 次に確認すること 3

## 入口

- NEXT_TASK.md
- 最新レポート
```

制約:
- `最新状態` は最大 4 行。
- `次に見る` は最大 3 行。
- `入口` は最大 2 リンク。
- 見出しは `最新状態`、`次に見る`、`入口` の 3 つだけ。
- 履歴、経緯、古い版の説明、長いリンク集、実施内容の詳細は書かない。
- 古い情報は追記で残さず、最新状態へ置き換える。
- 詳細は `📒打ち合わせノート.md`、`NEXT_TASK.md`、`履歴/progress.md` に分ける。

## 更新

- 次の判断が変わったら `NEXT_TASK.md`
- フェーズや大型節目が変わったら `開発ロードマップ.md`
- 新しい事実や実作業が出たら `履歴/progress.md`
- 節目完了、前提変更、ブロッカー変化のときだけ `履歴/スレッド引き継ぎファイル.md`
