# 定期レポート生成ガイド

更新日: 2026-05-26 JST

## 目的

日次の Markdown レポート群を、毎回手動でコマンドを打ち分けなくても同じ順序で生成できるようにする。

このガイドの対象はレポート生成だけです。実弾発注、取引所API送信、秘密鍵連携、自動 commit / push は対象外です。

## 主系コマンド

```bash
.venv312/bin/python tools/run_daily_reports.py
```

## よく使う実行例

通常実行:

```bash
.venv312/bin/python tools/run_daily_reports.py
```

日付指定の dry-run:

```bash
.venv312/bin/python tools/run_daily_reports.py --date 20260526 --dry-run
```

軽い日次更新だけ回す:

```bash
.venv312/bin/python tools/run_daily_reports.py --date 20260526 --skip-heavy --skip-ai
```

## 生成対象

通常実行では次を順に呼ぶ。

1. `daily-sync`
2. `paper_opportunity_diagnostics`
3. `market_map_effectiveness`
4. `operational_focus`
5. `paper_entry_sl_wait_redesign`
6. `report_hub`

`paper_entry_sl_wait_redesign` 専用 CLI が存在しない場合は、既存の paper diagnostics CLI を再利用して出力だけ切り替える。

## ログ

結果は `logs/runtime/` に保存する。

- `daily_reports.out`
- `daily_reports.err`
- `daily_reports_last_result.json`

レポート生成に失敗しても、監視本体は止めない。

## 自動化の考え方

- GitHub への自動 commit / push はまだしない
- launchd / cron へ入れる場合も、まずはローカル実行で安定を確認する
- 推奨時刻は JST 06:10 前後

## 補足

- 最新の作業ブランチは常に `運用資料/NEXT_TASK.md` を正本とする
- ブランチを切った場合は、同じ作業単位で `NEXT_TASK.md` を更新する
