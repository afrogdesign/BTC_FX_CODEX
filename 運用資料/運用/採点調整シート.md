# 採点調整シート

このシートは、通知や判定が慎重すぎる・攻めすぎると感じたときに、毎回コードを探し回らずに調整ポイントをすぐ確認するための早見表です。

## 今回の変更

- `CONFIDENCE_LONG_MIN`: `65 -> 40`
- `MIN_RR_RATIO`: `1.3 -> 1.15`
- `sweep_incomplete` の位置リスク加点: `+8 -> +4`

## まず見る場所

- 時系列の結果確認: `logs/csv/trades.csv`
- 各回の詳細確認: `logs/signals/*.json`
- 閾値の設定: `config.py`
- 実運用の上書き値: `.env`
- 位置リスクの採点: `src/analysis/position_risk.py`
- RR とセットアップ有効/無効: `src/analysis/rr.py`
- confidence の採点: `src/analysis/confidence.py`
- 通知の最終ゲート: `src/notification/trigger.py`

補足:
`config.py` の値は既定値です。実際に走る値は `.env` があればそちらが優先されます。

## 主要パラメーター一覧

| 項目 | 現在値 | 役割 | 主な影響先 | 定義場所 |
| --- | --- | --- | --- | --- |
| `CONFIDENCE_LONG_MIN` | `40` | ロング通知を出す最低 confidence | 低すぎると通知増、高すぎると初動を逃す | `config.py` |
| `CONFIDENCE_SHORT_MIN` | `70` | ショート通知を出す最低 confidence | 今回は未変更 | `config.py` |
| `MIN_RR_RATIO` | `1.15` | 最低損益比率。下回ると `RR_insufficient` | セットアップ無効、confidence にも間接影響 | `config.py` / `src/analysis/rr.py` |
| `sweep_incomplete` 加点 | `+4` | 流動性掃除未完了の位置リスク | `prelabel` が `SWEEP_WAIT` へ寄りやすくなる | `src/analysis/position_risk.py` |
| `POSITION_RISK_MEDIUM_THRESHOLD` | `45` | `SWEEP_WAIT` に上がる境界 | 位置リスクが待機へ倒れる強さ | `config.py` |
| `POSITION_RISK_HIGH_THRESHOLD` | `70` | `NO_TRADE_CANDIDATE` に上がる境界 | 強い見送り判定 | `config.py` |
| RR不足の confidence 減点 | `< 1.3` で `-15` | confidence を直接下げる | 通知抑制に効く | `src/analysis/confidence.py` |
| `CONFIDENCE_ALERT_CHANGE` | `10` | 前回通知から confidence 差で通知する閾値 | 通知頻度 | `config.py` |

## 今回のケースで効いていた観点

- `near_resistance_penalty`
  - 上値が近いとロングの加点を打ち消しやすい
- `rr_long_penalty`
  - 上がり始めでも RR が低く見積もられると止まりやすい
- `sweep_incomplete`
  - 初動で出やすく、待機判定へ寄せやすい
- `confidence_below_long_min`
  - ロング寄りでも通知を止める最後のゲート

## 調整するときの見方

### 1. チャンスを逃しすぎるとき

- `CONFIDENCE_LONG_MIN`
- `MIN_RR_RATIO`
- `sweep_incomplete` 加点
- RR不足の confidence 減点

### 2. 無駄な通知が増えすぎたとき

- `CONFIDENCE_LONG_MIN`
- `POSITION_RISK_MEDIUM_THRESHOLD`
- `POSITION_RISK_HIGH_THRESHOLD`
- `CONFIDENCE_ALERT_CHANGE`

## 判断の目安

- 「方向は合っているのに通知が出ない」
  - `src/notification/trigger.py`
  - `CONFIDENCE_LONG_MIN`
- 「bias は long なのに setup が invalid/watch のまま」
  - `src/analysis/rr.py`
  - `src/analysis/position_risk.py`
- 「confidence が低すぎて全然届かない」
  - `src/analysis/confidence.py`
- 「初動で毎回 Sweep待ちになる」
  - `src/analysis/position_risk.py`

## 最小確認手順

1. `logs/csv/trades.csv` で対象時刻の `bias / confidence / prelabel / no_trade_flags / suppress_reason_codes` を見る
2. `logs/signals/<signal_id>.json` で `score_factor_breakdown_*` と `risk_flags` を見る
3. このシートで該当パラメーターの定義場所を確認する
4. 必要最小限だけ調整する

## 補足

- いまの設計は「方向感」と「今入ってよいか」を別で見ています。
- そのため、`bias=long` でも通知が出ないことがあります。
- 初動をもっと拾いたい場合は、`confidence` と `位置リスク` と `RR` の3点を一緒に見るのが近道です。
