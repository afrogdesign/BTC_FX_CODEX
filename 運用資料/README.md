# 運用資料 入口

更新日: 2026-03-23 17:45 JST

このフォルダは、BTC Monitor の記録と判断資料の入口です。
迷ったときは次の順で見ます。

## AI の入口

1. `NEXT_TASK.md`
2. `開発ロードマップ.md`
3. 足りないときだけ `スレッド引き継ぎファイル.md`
4. 実施履歴が必要なときだけ `progress.md`

## 人の入口

- 直近の状況確認: `運用/README.md`
- 日々の要点確認: Obsidian 側 `👩‍⚖️秘書.md`
- 作業の流れ確認: Obsidian 側 `📒打ち合わせノート.md`

## ファイルの役割

- `NEXT_TASK.md`
  - 次の AI が今すぐ判断に使う短い実務メモ
- `開発ロードマップ.md`
  - 現在フェーズと次の大型節目だけを見る 2 枚目の入口
- `progress.md`
  - 実施履歴の入口。古い履歴は `progress_weekly/` へ退避
- `スレッド引き継ぎファイル.md`
  - 通常入口だけでは迷うときにだけ使う例外メモ
- `計画/`
  - フェーズ別計画や評価仕様などの中期計画
- `運用/`
  - 運用ルール、環境メモ、日常手順、調整資料
- `参考資料/`
  - すぐ読まない補助資料や過去検討メモ

## 今回の整理方針

- Mac 横断の環境一覧は `Global_BOX` を正本にする
- このフォルダには BTC 固有の判断と履歴だけ残す
- 入口ファイルに時系列の重い経緯を持ち込まない
